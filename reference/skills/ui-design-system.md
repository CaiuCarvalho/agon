# Skill: UI Design & Consistência

## Objetivo
Manter qualidade visual e consistência de UX sem quebrar padrões existentes do projeto.

## Fontes obrigatórias
- `reference/frontend.md`
- `reference/design/design-system.md`
- `reference/design/ui-principles.md`

## Regras de design
- Reutilizar componentes e padrões já existentes antes de criar novos.
- Garantir contraste e legibilidade em estados normais/erro/loading.
- Preservar consistência de espaçamento, tipografia e feedback visual.
- Responsividade obrigatória (mobile-first).

## Regras de implementação UI
- Não misturar regra de negócio complexa no componente de apresentação.
- Estados de loading/empty/error são obrigatórios em telas de dados.
- Formulários com validação clara e mensagens acionáveis.
- Acessibilidade mínima: foco visível, labels, navegação por teclado.

## Checklist de entrega UI
- [ ] Fluxo completo sem quebra visual.
- [ ] Componentes reutilizados quando possível.
- [ ] Estados de erro e carregamento implementados.
- [ ] Mobile validado.
